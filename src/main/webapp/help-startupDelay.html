This determines how long the Android Emulator plugin should delay for, at the start of a build, before attempting to launch the emulator.

This is useful, for example, if starting the emulator depends on other Build Environment plugins that need time to fully start up.
e.g. If you find that starting a VNC server using the Xvnc plugin can take 30 seconds — due to slow hardware, or a large X startup script — you would enter 30 (or more) here.
